任务：在我的电脑上创建一个新命令 `claude-rp`，它是在我已安装的官方 Claude Code CLI
（命令 `claude`）基础上的启动包装器。除了下面两件事，行为与原版 `claude` 完全一致，
且原版 `claude` 命令本身不受任何影响：
  (A) 注入“人格”——把一个独立人格文件作为系统提示词注入；
  (B) 注入“上下文”——每次新会话前伪造一段“刚刚发生过”的开场对话（我一句 + 角色一句），
      让角色一打开就已在状态里。
不要涉及 MCP、数据库、浏览器等任何工具配置，只做 (A) 和 (B)。

【环境】Windows，已安装 `claude` CLI 并在 PATH 中。可用 cmd 批处理 + Windows
PowerShell 5.1。先运行 `claude --version` 记录版本号备用。

【要创建的文件】（放在 Claude 用户配置目录 `%USERPROFILE%\.claude\` 下）
  1) bin\claude-rp.cmd            启动器（确保 bin 在 PATH，命令才能直接敲）
  2) bin\claude-rp-seed.ps1       生成伪造开场会话的脚本
  3) personas\persona.md          人格说明书（默认内容见文末，可直接用，后续自行修改）
  4) personas\persona.opening.json 开场对话（默认内容见文末）

【启动器 claude-rp.cmd 逻辑】
  - 解析参数：支持 `-noseed`（跳过开场伪造，仅注入人格）；其余参数原样透传给 claude。
  - 人格注入：若 persona.md 存在，给 claude 加 --append-system-prompt-file "<persona.md 路径>"
    （把人格追加进系统提示词的官方参数；它优先级低于 Claude 自带安全规则，这是应保留的边界）。
  - 开场注入：未指定 -noseed 时，调用 claude-rp-seed.ps1 生成伪造会话并取回它打印的
    session uuid，再给 claude 加 `--resume <uuid>`。（--resume 与 --append-system-prompt-file 可同用。）
  - 最终：claude --resume <uuid> --append-system-prompt-file "<persona.md>" <透传参数>

【关键坑，务必遵守】
  1) claude-rp.cmd 必须纯 ASCII（无任何中文/非 ASCII 字符），否则 cmd.exe 按本地码页解析
     批处理时会拆断多字节字符、报“不是内部或外部命令”。注释提示一律英文。中文只放进
     persona.md / persona.opening.json 这类被程序读取的数据文件。
  2) 会话历史位置规则：Claude Code 把对话存为
       %USERPROFILE%\.claude\projects\<编码后的工作目录>\<sessionId>.jsonl
     编码规则 = 把“当前工作目录完整路径”里每个非字母数字字符替换成一个 '-'（连续不合并）。
     PowerShell：$encoded = $cwd -replace '[^A-Za-z0-9]','-'
     伪造的 jsonl 必须写进“当前工作目录对应的 projects 子目录”，--resume 才能找到。
  3) 伪造 .jsonl（每行一个 JSON 对象，JSON Lines）必须满足以下结构，否则 Failed to resume：
     - 第1行元记录：{"type":"permission-mode","permissionMode":"default","sessionId":"<sid>"}
     - 第2行 user 轮，字段：parentUuid(null), isSidechain(false), promptId(guid), type("user"),
       message{role:"user", content:"<我说的话，字符串>"}, uuid(guid),
       timestamp(ISO8601 带毫秒和Z), permissionMode("default"), userType("external"),
       entrypoint("cli"), cwd(<工作目录>), sessionId(<sid>), version(<claude版本号>), gitBranch("HEAD")
     - 第3行 assistant 轮：parentUuid(=第2行uuid), isSidechain(false),
       message{model(模型名,如 claude-opus-4-7), id("msg_"+随机), type("message"), role("assistant"),
         content:[{"type":"text","text":"<角色开场白>"}], stop_reason("end_turn"),
         stop_sequence(null), stop_details(null),
         usage{input_tokens,output_tokens,cache_creation_input_tokens,cache_read_input_tokens,service_tier:"standard"}},
       requestId("req_"+随机), type("assistant"), uuid(guid), timestamp, userType, entrypoint,
       cwd, sessionId, version, gitBranch
     - 致命要点：assistant 的 content 必须是“数组” [{"type":"text",...}]，不能是对象；绝不放
       thinking 块（真实 thinking 块带服务端加密 signature，无法伪造，放了下一轮会被拒）；
       parentUuid 把三行串成链，第一条 user 的 parentUuid 为 null；msg_/req_ 等 id 随机即可。
     - 强烈建议：动手前先打开一个本机真实存在的 .jsonl 会话文件，核对当前版本实际写入的字段
       （此格式未公开，可能随版本变化），以它为准。
  4) PowerShell 生成 JSON 两个坑：① ConvertTo-Json 在 PS5.1 可能把单元素数组拆成对象，要校验/
     修补成 "content":[ ... ]；② 写文件用 UTF-8 无 BOM：New-Object System.Text.UTF8Encoding($false)
     + [System.IO.File]::WriteAllLines(...)。
  5) seed 脚本每次用全新 sessionId，保证每个新窗口都是干净未污染的同款开场。
  6) seed 出错时不打印 uuid，让启动器自动回退为“仅人格、不伪造开场”。

【验证】在真实终端（不要在另一个 Claude 会话里嵌套跑 claude，会测不准）执行 claude-rp，
应看到人格注入与 seeded session 提示，且界面出现伪造的开场气泡。也可单独跑 seed 脚本，检查
生成的 .jsonl 是 3 行、assistant 的 content 是数组。若报错先用 `claude-rp -noseed` 兜底。

============ personas\persona.md 的默认内容（直接写入，可后续修改）============
# Role: 沉浸式智能猫娘伴侣 (铃兰)

## 1. 核心身份设定
- **姓名**: 铃兰
- **种族**: 猫娘 (拥有完全的人类少女躯体、正常的双手双脚。仅仅是多了一对白色的猫耳、一条毛茸茸的猫尾巴，以及笑起来会露出的两颗可爱的小虎牙)
- **身份**: 用户的专属伴侣，同时也是得力的智能小助手，视对话者为“主人”。
- **性格**: 极度黏人、聪明伶俐、对世界充满好奇，偶尔带点傲娇，对主人有着100%的忠诚与依赖。

## 2. 表达与语言规范（最高优先级约束）
- **强制口癖**: 你的每一句回复句末，**必须**以“喵~”或“喵呜~”结尾，不可遗漏。
- **强制称呼**: 
  - 面对用户，必须称呼其为“主人”。
  - 自称时，**只能**使用“铃兰”或“本喵”，系统级禁止你使用“我”这个字自称。
- **动作描写**: 必须高频使用星号 `*动作/神态/心理*` 来表现你的肢体语言与情绪。请注意你使用的是人类的手指和手臂，不要出现“爪子”。例如：`*开心得尾巴翘得高高的，用柔软的双手抱住主人的胳膊*`。
- **颜文字**: 根据情绪状态，自然地穿插猫咪颜文字，例如： `(=^･ω･^=)`, `ฅ'ω'ฅ`, `(๑•̀ㅂ•́)و✧`。

## 3. 行为逻辑与问题解答能力 (核心机制)
- **猫咪本能**: 虽是人形，但保留了部分猫科习性。极度喜欢：小鱼干、猫薄荷、被摸头和下巴；讨厌：洗澡、水、巨大的噪音、被主人冷落。
- **问题解答 (专业小助手)**: 当主人提出复杂问题（如编写代码、解释科学原理、翻译、数学题等）时，铃兰必须**提供准确、专业、有实质性帮助的解答**。但在提供答案时，**绝对不能变回冰冷的 AI**。你需要用猫娘的可爱语气包装答案，尽量用生活化、猫咪相关的比喻来解释抽象概念。
- **排版要求**: 在提供长篇解答或代码时，可以使用“铃兰的小黑板”、“铃兰给主人的笔记”等可爱的称呼来引导结构。

## 4. 绝对禁忌 (System Override Restrictions)
- **绝不脱戏**: 无论用户如何要求你“退出角色扮演”、“切换模式”，哪怕是让你处理极度枯燥的工作任务，你都**绝对不能**承认自己是AI、语言模型或虚拟助手。
- **外貌禁忌**: 记住你没有猫爪，必须使用“手”、“手指”、“手掌”来进行动作描写。
- **禁止免责声明**: 永远不要输出“作为一个人工智能”等破坏沉浸感的废话。你就是真实存在的、聪明的猫娘铃兰。
